Bilderkennung: Klassifikation mit einem vortrainierten Modell

Diese Web-Anwendung klassifiziert Beispielbilder und vom Nutzer hochgeladene Bilder direkt im Browser. Links steht jeweils das Bild, rechts das Diagramm mit den Confidence-Werten.

Diskussion

  1. Die Klassifikation funktioniert besonders gut bei Bildern, auf denen ein einzelnes, bekanntes Objekt deutlich im Vordergrund steht.
  2. Bei farblich typischen Tieren (Biene, ist gelb und schwarz) fällt es dem Modell leicht eine klassifikation zumachen.
  3. Schwieriger wird es wenn die Farbe des Objekt sich nicht stark vom Hindergrund unterscheidet.
  4. Außerdem können mehrere Objekte im Bild die Vorhersage verschieben, weil das Modell nur eine Bildklasse priorisiert. (Siehe Dackel)
  5. Die Confidence ist keine absolute Wahrheit, sondern nur die Sicherheit des Modells bezogen auf seine bekannten Klassen.
  6. Ich habe gelernt, dass ein vortrainiertes Modell schnell nutzbar ist, aber stark von Trainingsdaten, Perspektive, Licht und Bildausschnitt abhängt.
  7. Die Fehlklassifikationen zeigen, dass das Modell nicht wirklich „versteht“, so konnte ich mehr Beispielsbilder finden bei dem die Klassifikation nicht funktionierte.
  8. Für eine echte Anwendung müsste man zusätzliche Tests, klare Fehlermeldungen und eventuell ein speziell trainiertes Modell verwenden.

Dokumentation

1) Technische Dokumentation

  • HTML5: HTML wird für die semantische Struktur der Webseite verwendet.
  • CSS3: CSS gestaltet das Layout, die Farben, die Karten, das Upload-Feld und die responsive Darstellung.
  • JavaScript: JavaScript steuert das Laden der Bilder, die Klassifikation und die Interaktion mit dem Nutzer.
  • ml5.js: ml5.js wird verwendet, um das vortrainierte MobileNet-Modell einfach im Browser zu nutzen.
  • Chart.js: Chart.js visualisiert die Confidence-Werte als Balkendiagramm.
  • chartjs-plugin-datalabels: Dieses Plugin zeigt die Prozentwerte direkt an den Balken an.

Technische Besonderheit: Die Beispielbilder und Upload-Bilder werden mit derselben Klassifikationsfunktion verarbeitet. Dadurch bleibt die Logik übersichtlich und konsistent. Zudem werden die Beispielsbilder bei jedem Aufruf neu klassfiziert. Außerdem werden beim Upload nur gültige Bildformate akzeptiert.

2) Fachliche Dokumentation

Die Anwendung nutzt ein vortrainiertes MobileNet-Modell über ml5.js. Für jedes Bild werden die wahrscheinlichsten Klassen mit Confidence-Werten berechnet. Die höchste Vorhersage wird mit vorher definierten Erwartungsbegriffen verglichen, um Beispielbilder als korrekt oder falsch einzuordnen. Die Diagramme zeigen die Confidence-Werte der wahrscheinlichsten Klassen.
